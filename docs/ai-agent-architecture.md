# AI 에이전트 기반 실시간 트레이딩 봇 — 설계 문서

**작성일:** 2026-04-29  
**버전:** v0.1 (초안)

---

## 1. 목표

- 한국 주식 시장(KOSPI/KOSDAQ)에서 AI 에이전트가 실시간 시세를 보고 매매를 판단
- 판단 근거(reasoning)를 대시보드에 즉시 반영
- 모의투자 → 실투자 전환이 가능한 구조

---

## 2. AI 모델 선정

트레이딩 에이전트는 **판단 속도**와 **판단 품질**을 동시에 요구한다.  
단일 모델이 아닌 **역할별 분리(Multi-tier)** 구조를 권장한다.

### 2.1 모델 비교

| 모델 | 공급사 | 평균 응답속도 | 강점 | 약점 |
|------|--------|-------------|------|------|
| **claude-haiku-4-5** | Anthropic | ~0.5초 | 빠름, Tool use 안정적, 저비용 | 복잡한 추론 한계 |
| **claude-sonnet-4-6** | Anthropic | ~1.5초 | 추론 품질 우수, Tool use 강력 | Haiku보다 느림 |
| **gemini-2.0-flash** | Google | ~0.4초 | 최고 속도, 멀티모달 | Tool use 일관성 다소 낮음 |
| **gpt-4o-mini** | OpenAI | ~0.6초 | 범용성, 생태계 풍부 | 한국어 금융 컨텍스트 약함 |
| **Qwen2.5-7B (로컬)** | Alibaba | ~0.3초 | 완전 무료, 네트워크 불필요 | GPU 필요, 설정 복잡 |

### 2.2 권장 구성: Dual-tier 에이전트

```
[실시간 시세 tick]
       ↓
 ┌─────────────────────────────┐
 │  Tier 1: 즉각 반응 에이전트  │  ← claude-haiku-4-5
 │  - 매 30초~1분 판단          │     or gemini-2.0-flash
 │  - BUY / SELL / HOLD 결정   │
 │  - 신뢰도 점수 출력          │
 └─────────────┬───────────────┘
               │ 신뢰도 낮음 or 고액 주문
               ↓
 ┌─────────────────────────────┐
 │  Tier 2: 심층 분석 에이전트  │  ← claude-sonnet-4-6
 │  - 필요 시에만 호출          │
 │  - 시장 맥락, 뉴스 종합      │
 │  - 최종 승인/거부 판단       │
 └─────────────────────────────┘
```

**Tier 1 기본 추천: `claude-haiku-4-5-20251001`**
- Anthropic Tool use 스펙을 완전히 따르므로 `buy()` / `sell()` 툴 실행이 안정적
- 토큰 비용이 낮아 분당 수십 회 호출해도 경제적
- 응답에 reasoning 필드를 포함시켜 대시보드에 판단 근거 표시 가능

**Tier 2: `claude-sonnet-4-6`**
- Tier 1이 신뢰도 70% 미만을 반환하거나 거래 금액이 임계치 초과 시에만 호출
- 비용 절감 + 속도 유지

## 4. AI 에이전트 설계 상세

### 4.1 Tool 정의

```python
tools = [
    {
        "name": "buy",
        "description": "지정 종목을 시장가로 매수한다. 확신이 있을 때만 호출할 것.",
        "input_schema": {
            "type": "object",
            "properties": {
                "ticker":     {"type": "string", "description": "종목코드 (예: 005930)"},
                "quantity":   {"type": "integer", "description": "매수 수량"},
                "confidence": {"type": "number", "description": "판단 신뢰도 0~100"},
                "reason":     {"type": "string", "description": "매수 근거 (한국어, 2~3문장)"},
            },
            "required": ["ticker", "quantity", "confidence", "reason"],
        },
    },
    {
        "name": "sell",
        "description": "지정 종목을 시장가로 매도한다.",
        "input_schema": {
            "type": "object",
            "properties": {
                "ticker":     {"type": "string"},
                "quantity":   {"type": "integer"},
                "confidence": {"type": "number"},
                "reason":     {"type": "string"},
            },
            "required": ["ticker", "quantity", "confidence", "reason"],
        },
    },
    {
        "name": "hold",
        "description": "현재 포지션을 유지한다. 불확실하면 항상 hold를 선택할 것.",
        "input_schema": {
            "type": "object",
            "properties": {
                "reason": {"type": "string"},
            },
            "required": ["reason"],
        },
    },
]
```

### 4.2 시스템 프롬프트

```
당신은 한국 주식 시장 전문 트레이딩 에이전트입니다.

규칙:
1. 제공된 OHLCV, 기술 지표, 포트폴리오 현황을 종합적으로 분석하세요.
2. 반드시 buy / sell / hold 중 하나의 툴을 호출하여 결정을 내리세요.
3. confidence가 60 미만이면 반드시 hold를 선택하세요.
4. 단일 종목에 포트폴리오의 30% 이상을 투자하지 마세요.
5. reason은 구체적인 지표 수치를 포함한 한국어 2~3문장으로 작성하세요.
6. 뉴스가 부정적이면 매수 기준을 높이세요.
```

### 4.3 컨텍스트 구조 (에이전트 입력)

```python
context = f"""
## 종목: {name} ({ticker})

### 현재가
- 현재가: {current_price:,}원
- 전일 대비: {change_pct:+.2f}%

### 기술 지표 (최근 20일 기준)
- RSI(14): {rsi:.1f}  {'과매도' if rsi < 30 else '과매수' if rsi > 70 else '중립'}
- 볼린저: 상단 {upper:,} / 중선 {mid:,} / 하단 {lower:,}
- SMA5: {sma5:,}  SMA20: {sma20:,}
- 거래량: {volume:,} (20일 평균 대비 {vol_ratio:.1f}배)

### 포트폴리오 현황
- 총자산: {total_assets:,}원
- 예수금: {cash:,}원
- {ticker} 보유: {holding_qty}주 (평균가 {avg_price:,}원, 현재 손익 {pnl:+.1f}%)

### 최근 뉴스 (감성 분석)
{news_summary}

### 판단 기준
- 매수 가능 금액: {buy_limit:,}원 이하
- 최대 매수 수량: {max_qty}주
"""
```

## 5. 실시간 대시보드 설계

### 5.1 기술 스택

| 레이어 | 선택 | 이유 |
|--------|------|------|
| 백엔드 | FastAPI | WebSocket 내장, 비동기 지원 |
| 프론트엔드 | React + TradingView Lightweight Charts | 금융 차트 전문 라이브러리 |
| 실시간 통신 | WebSocket (JSON 이벤트) | 서버→클라이언트 push |
| DB | SQLite (단기) → PostgreSQL (확장) | 로컬 우선, 추후 확장 |

### 5.2 WebSocket 이벤트 스키마

```json

// AI 에이전트 판단
{
  "type": "agent_decision",
  "ticker": "005930",
  "action": "BUY",
  "quantity": 10,
  "confidence": 82,
  "reason": "RSI 28로 과매도 구간 진입. 볼린저 하단 이탈 후 반등 신호 포착. 거래량 평균 대비 2.1배 증가.",
```

## 6. 구현 단계별 로드맵

### Phase 1: 데이터 파이프라인 (1~2주)
- [ ] KIS API 계좌 개설 및 모의투자 API 키 발급
- [ ] WebSocket 실시간 시세 수신 구현 (`market_stream.py`)
- [ ] tick 데이터 → 1분봉 OHLCV 실시간 변환
- [ ] 기술 지표 실시간 계산 (RSI, 볼린저, SMA)

### Phase 2: AI 에이전트 엔진 (1~2주)
- [ ] Claude Haiku Tool use 연동 (`agent_engine.py`)
- [ ] 컨텍스트 빌더 구현 (`context_builder.py`)
- [ ] Tier 1/2 분기 로직 구현
- [ ] 판단 결과 DB 저장 및 로그 스키마 설계

### Phase 3: 안전장치 및 주문 실행 (1주)
- [ ] 리스크 가드 레이어 (`risk_guard.py`)
  - 일일 최대 손실 한도
  - 종목당 최대 비중
  - 연속 손실 시 자동 중지
- [ ] KIS REST API 주문 실행 연동
- [ ] 모의/실투자 모드 전환 스위치

### Phase 4: 실시간 대시보드 (2~3주)
- [ ] FastAPI 백엔드 + WebSocket 서버
- [ ] React 프론트엔드 + TradingView 차트

- [ ] 비상정지 버튼 및 수동 오버라이드

### Phase 5: 고도화 (지속)
- [ ] 뉴스 감성 분석 연동

## 7. 비용 추정 (Claude API)

| 시나리오 | 모델 | 호출 빈도 | 월 예상 비용 |
|---------|------|----------|------------|
| 감시 종목 5개, 1분 간격 | Haiku | 7,200회/일 | ~$5 |
| 감시 종목 10개, 30초 간격 | Haiku | 28,800회/일 | ~$15 |

> Sonnet은 고확신 판단이 필요한 경우에만 호출하여 비용 최소화.

## 8. 리스크 및 주의사항

1. **AI 환각(Hallucination)**: 에이전트가 잘못된 판단을 내릴 수 있음 → confidence 임계값 + 인간 승인 레이어 필수
2. **API 지연**: Claude API 응답이 느릴 경우 기회 손실 → timeout 설정 및 fallback(rule-based) 준비
3. **모의→실투자 전환**: 실투자 전 최소 1개월 모의 운용 후 성과 검증 필수
4. **장 시간 외 호출**: 한국 주식은 09:00~15:30만 유효, 에이전트 호출 스케줄 제한 필요
5. **법적 책임**: AI 판단으로 인한 손실은 전적으로 운용자 책임
